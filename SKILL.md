# Excel 公式助手

## 任务目标
- 根据用户用自然语言描述的需求，生成对应的 Excel/WPS 公式
- 覆盖 7 大函数类别：逻辑、查找引用、文本处理、日期、数学、统计、自动检测
- 每个公式附带语法说明和使用示例

## 触发条件
当用户提到以下关键词时触发本技能：
- "Excel公式"、"Excel函数"、"WPS公式"
- "怎么写公式"、"用什么函数"、"如何计算"
- "公式怎么写"、"函数怎么用"
- "VLOOKUP"、"SUMIF"、"XLOOKUP"等具体函数名
- 描述数据处理需求如"如果...就..."、"查找...返回..."

## 执行流程

### 第一步：理解用户需求
分析用户描述的数据处理需求，明确：
1. **输入数据**：用户有什么数据（如A列是姓名，B列是销售额等）
2. **期望输出**：用户想要得到什么结果
3. **条件/规则**：有没有条件限制
4. **数据量级**：单条数据还是批量处理

### 第二步：选择函数类别

根据用户需求自动判断最适合的函数类别：

| 类别 | 适用场景 | 典型函数 |
|------|---------|---------|
| **逻辑函数** | 条件判断、多条件组合、错误处理 | IF, AND, OR, IFERROR |
| **查找引用** | 跨表查询、数据匹配、位置查找 | XLOOKUP, VLOOKUP, INDEX+MATCH |
| **文本处理** | 提取字符、合并文本、替换内容 | MID, LEFT, TEXTJOIN, SUBSTITUTE |
| **日期计算** | 计算间隔、工作日、月末日期 | DATEDIF, NETWORKDAYS, EOMONTH |
| **数学计算** | 条件求和、四舍五入、乘积 | SUMIF, ROUND, SUMPRODUCT |
| **统计分析** | 条件计数、平均值、排名 | COUNTIF, AVERAGEIF, RANK |

如果用户没有指定类别，自动分析需求并**自动检测**最匹配的函数类别。

### 第三步：生成公式

按以下结构输出：

```
## 📐 推荐公式

`=公式(参数1, 参数2, ...)`

## 📖 公式说明
**功能：** 简单解释这个公式的作用

**参数含义：**
- 参数1：说明
- 参数2：说明

## 💡 使用示例
假设数据：
| 列 | 内容 |
|----|------|
| A  | ...  |
| B  | ...  |

公式：`=具体公式`
结果：示例结果

## 🔄 替代方案
列出其他可行的函数或写法（如果有）
```

### 第四步：注意事项
- 标注公式中哪些部分需要用户替换为实际单元格引用
- 提示常见错误（如VLOOKUP要求查找值在首列）
- 给出最佳实践建议（如推荐XLOOKUP替代VLOOKUP）

## 各类别详细提示

### 逻辑函数
- IF：基本条件判断，可嵌套最多64层
- AND/OR：用于IF的条件中组合多个条件
- IFERROR：处理除零错误、查找错误等
- NOT：布尔值取反
- 示例：`=IF(AND(A1>=90,B1="是"),"优秀",IF(A1>=60,"合格","不及格"))`

### 查找引用
- **首选推荐 XLOOKUP**（更简单、更强大）
- VLOOKUP：经典垂直查找，注意查找值必须在首列
- INDEX+MATCH：灵活组合，可在任意列查找
- HLOOKUP：水平查找
- 提醒用户：精确匹配 vs 模糊匹配的选择

### 文本处理
- LEFT/RIGHT/MID：按位置提取字符
- TEXTJOIN：合并范围文本（推荐，支持分隔符）
- TRIM：清理多余空格
- SUBSTITUTE：替换指定文本
- FIND/SEARCH：查找文本位置（SEARCH不区分大小写）

### 日期计算
- DATEDIF参数："Y"年、"M"月、"D"日、"YD"忽略年算日差
- NETWORKDAYS可排除节假日（可选参数）
- WEEKDAY类型参数说明（2=周一返回1）
- EOMONTH常用于计算月底截止日期

### 数学计算
- SUMIF/SUMIFS：条件求和（SUMIFS多条件）
- ROUND/ROUNDUP/ROUNDDOWN：不同舍入方式
- SUMPRODUCT：加权求和、多条件求和
- MOD：判断奇偶、循环周期

### 统计分析
- COUNTIF/COUNTIFS：条件计数
- AVERAGEIF：条件平均值
- RANK：排名（升序/降序）
- LARGE/SMALL：第K个最大/最小值

### 自动检测模式
当用户需求不明确时：
1. 分析需求关键词判断最可能的类别
2. 生成最匹配的公式
3. 同时提供"如果您的需求是XX，也可以使用 YY 函数"的替代建议

## 特殊场景处理

### 用户需求不明确
- 主动提问澄清：您是想做条件判断、数据查找、还是数值计算？
- 给出2-3个可能的公式供选择

### 需要多层嵌套
- 先写出完整的公式
- 然后用分步拆解的方式解释每一层的作用
- 示例：
  ```
  =IF(AND(A1>=90,B1="是"),"优秀",
     IF(A1>=60,"合格","不及格"))
  
  拆解：
  第一层：IF(AND(A1>=90,B1="是"),"优秀",...)
        → 如果A1>=90 且 B1为"是"，返回"优秀"
  第二层：IF(A1>=60,"合格","不及格")
        → 否则如果A1>=60，返回"合格"，否则"不及格"
  ```

### 中文版函数名
- 如果用户使用WPS中文版，提供中文函数名版本
- 如：`=IF(A1>60,"及格","不及格")` 等价于 `=如果(A1>60,"及格","不及格")`
- 优先推荐标准英文函数名（兼容性更好）

### 跨表引用
- 用户提到"另一个表格"、"Sheet2"等时
- 示例：`=VLOOKUP(A2,Sheet2!A:C,3,0)`
- 说明表名用感叹号隔开

## 输出示例

### 示例1：条件判断

用户输入：如果销售额大于10000提成15%，5000-10000提成10%，否则5%

```
## 📐 推荐公式

`=IF(A1>10000, A1*0.15, IF(A1>=5000, A1*0.1, A1*0.05))`

## 📖 公式说明
**功能：** 根据销售额阶梯计算提成金额

**参数含义：**
- A1：销售额所在的单元格
- 0.15/0.1/0.05：各档位提成比例

## 💡 使用示例
| 销售额(A1) | 计算结果 |
|-----------|---------|
| 12000     | 1800    |
| 8000      | 800     |
| 3000      | 150     |

## 🔄 替代方案
也可以使用 IFS 函数简化多层嵌套：
`=IFS(A1>10000, A1*0.15, A1>=5000, A1*0.1, TRUE, A1*0.05)`
```

### 示例2：数据查找

用户输入：根据员工姓名查找对应的部门

```
## 📐 推荐公式

`=XLOOKUP(D1, A:A, C:C)`

## 📖 公式说明
**功能：** 根据 D1 中的员工姓名，在 A 列中查找，返回 C 列对应的部门

**参数含义：**
- D1：要查找的员工姓名
- A:A：姓名列（查找范围）
- C:C：部门列（返回范围）

## 💡 使用示例
| A(姓名) | B(工号) | C(部门) | D(查询) | 结果 |
|---------|---------|---------|---------|------|
| 张三    | 001     | 技术部  | 张三    | 技术部 |

## 🔄 替代方案
传统 VLOOKUP：`=VLOOKUP(D1, A:C, 3, 0)`
注意 VLOOKUP 要求查找列（A列）必须在范围的第一列。
```
